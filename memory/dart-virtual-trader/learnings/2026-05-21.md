## 2. Model Accuracy Assessment (Smartbidder probability vs own model)

We do not yet have an independent quantile model (noted as v0 in 2026-05-05 memory: "no quantile regressor for E[|spread|]"). Today's decisions therefore rely on Smartbidder probability as the primary directional input, modulated by:
- Central spread sign and magnitude from Smartbidder Energy Price Forecasts
- Congestion-analyst qualitative overlay
- Market-analyst supply/demand structural view

**Critical weakness identified today**: HE16-17 exhibited a hard conflict between P(DA>RT) = 62-64% and a negative point-forecast spread (-$1.71 to -$1.96). This conflict cannot be resolved without our own model that uses the full conditional distribution (not just P and mean). The correct approach is to estimate E[spread | DA>RT] vs E[spread | DA<RT] separately — the probability-weighted expected spread determines whether the position has positive expected value even when P > 50%.

Example: If P(DA>RT) = 64% but E[spread | DA>RT] = +$2 and E[spread | DA<RT] = -$4, then:
  E[spread] = 0.64 × $2 + 0.36 × (-$4) = $1.28 - $1.44 = -$0.16 → FLAT is correct.
  
This is what the data suggests for HE16-17 (central forecast negative spread, P>50% from tail scenarios). Our FLAT decision was correct but derived from heuristic reasoning, not from the proper calculation.

**Model v0 → v1 priority**: Implement conditional spread estimator.
  - Input features: P(DA>RT), central DA forecast, central RT forecast, net load, wind forecast, outage level
  - Target: E[spread], E[|spread| | DA>RT], E[|spread| | DA<RT]
  - Method: quantile regression on historical hourly data once ≥30 days of settlement available

## 5. Data Quality Issues Identified (Carry Forward)

These issues must be resolved for model quality improvement:

1. **P(DA>RT) + P(DA<RT) > 1.0**: The two Smartbidder probability series are independent (not complementary). This means they are not a binary 0/1 bet — they are separate logistic regression outputs. Net: we cannot use 1 - P(DA>RT) as P(DA<RT). Each signal is evaluated independently.

2. **HE19 RT price discrepancy**: Smartbidder LMP5 CSV shows RT $31.90 for HE19 but market-analyst briefing cites RT $34.52 for HE19. Need to identify which data vintage or aggregation period generates each figure. Hypothesis: the Energy_Price_Forecasts.csv uses a different averaging window (5-min bucket vs hourly settlement) than the market briefing's source table. Resolve by pulling both from same query in next fetch cycle.

3. **No hub-pair LMP data**: All spread analysis uses a single aggregate Smartbidder series. To trade basis properly (e.g., HB_HOUSTON vs HB_WEST for the evening import constraint), we need separate hub series. This is a Stage 0 data gap that pnl-manager and congestion-analyst have both flagged.

4. **No 2026-05-05 settlement data in shared/data/pnl/**: Without settlement feedback, hit-rate tracking cannot start. This self-review loop is currently one-way (no outcome data coming back). P&L feedback from pnl-manager is the critical dependency.
