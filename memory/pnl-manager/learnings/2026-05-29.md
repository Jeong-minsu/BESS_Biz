# PnL Manager — Daily Learnings 2026-05-29

## What was compiled

GKS actual P&L for flowday 2026-05-29 from Tenaska PTP Battery-Settlement-Details (ESR UUID), compared against Smartbidder "Mount Blue Sky with Virtuals (RTC Version)" benchmark.

## Key figures

- GKS Total: +$5,111.74
- Benchmark Total: +$3,970.21
- Outperformance: +$1,141.53 (+28.8%)

## Data quality issues encountered

1. `2026-05-29_hsl.json` does not exist on disk. The HSL slot was listed in the fetch summary's `tenaska_endpoints_used` dict but the file was not written. This is a gap — HSL availability cannot be reported. Flagged in report and JSON.

2. `2026-05-29_da_energy_bid.json` does not exist on disk. The summary file shows the DA Energy Bid endpoint but no file was written. This is expected behavior (GKS bids submitted via Smartbidder), but the missing file should be noted so evaluator can distinguish "expected absent" from "unexpected missing."

3. Tenaska `RT_Energy_Amt` is a net line that bundles RT energy dispatch settlement with any RT AS deployment imbalances. The benchmark breaks these out separately (RT Non-Spin, RT RRS, RT ECRS). Direct per-product comparison at the RT level is not possible without a finer Tenaska decomposition. This is a structural data limitation, not a one-day anomaly.

## Process observations

- The energy_as_detail.json spans both the flowday proper (05:00Z to 05:00Z+1) and some next-day hours; filtering to the correct UTC window is required. The file contained 376 records; 24 records per datapoint across 24 hours = clean.
- DA_RRS_Amt was zero for all HE01-HE20 and non-zero only in HE21-HE24. This matches the overnight RRS dispatch pattern visible in the Gen_RRS_Qty column (70 MW x 4 hours).
- DA_NS_Amt was non-zero HE01-HE13 (morning/midday) and zero from HE14 onward, with no NS in the evening. This means the NonSpin clearing was concentrated in the off-peak charging window.
- HE10 and HE11 show large negative RT_Energy_Amt (-$1,334.19, -$515.63). These are the biggest loss hours, coinciding with a large RT consumption event (RT_Consumption_Qty: 67.5 MWh and 25.8 MWh respectively in HE10-11), likely a dispatch or charging event at high RT prices.
- HE21 and HE22 show very large positive DA_Energy_Amt (+$2,522 and +$2,554.50). These correspond to 50 MWh DA_Sales_Qty in each hour (overnight generation sold at $50.44/$51.09 DASPP). Strong overnight DA discharge performance.

## Decisions and accuracy

- DART virtual revenue was set to null in the output JSON. `DA_Energy_Amt` captures the net DA settlement but cannot be cleanly attributed to "DART virtual" without the bid/offer decomposition. The summary quick-stats did not include a DART Virtual line. This is the correct conservative approach.
- Smartbidder benchmark "Total" column was used directly as the benchmark reference. The cumulative EOD figure ($3,970.21) matches the stated benchmark figure exactly.

## For future improvement

- Investigate whether HSL data can be retrieved separately or whether the HSL slot in the Battery-Settlement-Details query is unreliable for this ESR UUID.
- Consider requesting Tenaska to break out RT Non-Spin, RT RRS, and RT ECRS imbalance settlements as separate columns in the Battery-Settlement-Details response, to enable product-level RT comparison with Smartbidder.
